# 06. Settlement Process Note

## 문서 목적

이 문서는 legacy settlement 흐름을 참고용으로만 정리한 노트다.
현재 bootstrap의 `SettlementRun` / `SettlementItem`은 이 문서의 계산 엔진을 구현하지 않으며, 단순한 placeholder scaffold로만 유지한다.

## 입력 소스 예시

레거시 정산은 대체로 아래 종류의 입력을 묶어서 처리했다.

- 기사 기본 정보와 소속 정보
- 운행/배송/출근성 로그
- 정산 정책과 기본 단가
- 추가 정산/보정 항목
- 청구, 공제, 차량 임대료, 외주 이력

이 문서는 각 입력의 의미만 남기고, 개별 모델이나 계산식을 이식하지 않는다.

## 일일 집계 단계

일 단위로는 당일 발생한 운영 이벤트를 모아 기사별 요약치를 만드는 흐름이 있었다.

- 수행 건수와 기본 매출성 항목을 모은다
- 보정이 필요한 건을 임시 항목으로 남긴다
- 일별 결과를 다음 단계의 중간 집계로 전달한다

이 단계는 레거시 용어의 `DailySettlement`를 떠올리기 위한 참고만 제공한다.

## 월별 집계 단계

월 단위로는 일일 집계 결과를 기간 기준으로 다시 합산했다.

- 기사별 월 누적액을 계산한다
- 확정 전후 상태를 분리해 관리한다
- 월말 마감용 결과를 생성한다

이 단계는 `MonthlySettlement`라는 레거시 용어를 이해하기 위한 참고다.

## 보험 / 세금 / 공제 단계

레거시 흐름에는 정책 기반 차감 단계가 별도로 있었다.

- 보험성 차감
- 세금성 차감
- 기타 공제와 조정

이 문서는 어떤 비율, 공식, 순서를 써야 하는지 정의하지 않는다.

## 이번 bootstrap에서 하지 않는 것

이번 bootstrap에서는 다음을 구현하지 않는다.

- 정산 계산 엔진
- 정책/설정/비율 테이블
- 일별/월별/그룹 정산 clone 모델
- 보험료율, 세율, 공제 공식
- Driver master 직접 FK 결합

현재 서비스는 `SettlementRun` / `SettlementItem` 구조만 유지하는 placeholder다.
